# BOSS CARGO EXPRESS AI - TRAINING PIPELINE

This directory contains the organized training data and configuration for the Boss Cargo Express AI. The hierarchy is designed to be linear, matching the structure of the compiled `Modelfile`.

## 1. Directory Structure

### `sources/` - **The Source of Truth**
*   **`prompts/`**: Modular system instructions, numbered by their order of appearance in the final prompt.
    1.  `01_persona.txt`: Identity, Culture (Boss), and Brand Voice.
    2.  `02_safety.txt`: Security, Privacy, and Scope Enforcement.
    3.  `03_hallucination.txt`: Factual Accuracy and Source Discipline.
    4.  `04_language.txt`: Bilingual rules.
    5.  `05_examples.txt`: Few-shot behavior examples.
*   **`knowledge/`**: Ground truth data for the AI.
    *   `01_raw/`: Original source documents.
    *   `02_processed/`: Refined, compressed, and time-stamped data.

### `config/` - **Build Configuration**
*   `allowed_sources.json`: Defines which processed files are included in the `KNOWLEDGE BASE`.
*   `model_config.json`: (Optional) Advanced training and prompt order settings.

### `build/` - **Generated Artifacts**
*   `Modelfile`: The final Ollama configuration (generated by `ai:train`).
*   `full_system_prompt.txt`: The compiled system prompt for review.
*   `full_knowledge_base.txt`: The compiled knowledge base for review.

## 2. Maintenance Workflow

1.  **Modify Sources**: Always edit files in `sources/prompts/` or `sources/knowledge/` first.
2.  **Train AI**: Run `sail artisan ai:train` to recompile the `Modelfile` and update the Ollama model.
3.  **Review Build**: Check the `build/` directory to verify the compiled output.
